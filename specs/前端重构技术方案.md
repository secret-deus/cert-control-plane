# 前端重构技术方案

## 需求概要

按照 `request.md` 的设计规范，重构前端为「证书资产管理 + 分发 + 监控」平台，核心页面：监控聚合面板、证书面板、Agent 面板。

## 关键技术决策

| 决策项 | 方案 | 理由 |
|--------|------|------|
| UI 框架 | 继续用 Tailwind CSS v4 + lucide-react，不加组件库 | 现有 14 个文件全是手写 Tailwind，加组件库会制造风格分裂 |
| 图表库 | 继续用 recharts | 现有已用且需求中趋势图 recharts 完全胜任 |
| 状态管理 | 不加 pinia/redux，保持 useState + useEffect | 项目体量不大，无需引入 |
| 数据获取 | 不加 React Query，保持 apiFetch | 同上，N+1 问题通过后端接口优化解决 |
| 路由结构 | 4 个主页面 + 1 个设置预留 | 对齐需求文档的 4 模块 |
| 删除策略 | 删除所有现有组件，完全重写 | 现有页面与需求差距过大，修补比重写成本更高 |

## 新页面结构

```
/ (AuthScreen)
/dashboard          → 监控聚合面板 (Dashboard)
/certificates        → 证书管理 (CertificatesPage)
/agents              → Agent 管理 (AgentsPage)
```

侧边栏从 6 项精简为 4 项，对齐需求文档。

## 后端 API 差异

现有 API 完全满足新前端需求，无需新增接口。数据映射关系：

| 前端需求 | 现有 API | 备注 |
|----------|----------|------|
| 证书总数 | `GET /dashboard/summary` | `certificates.total_active` |
| 7天内过期 | `GET /dashboard/cert-alerts` | `summary.external.critical + expired` |
| 30天内过期 | `GET /dashboard/summary` | `certificates.expiring_soon` |
| 异常节点 | `GET /dashboard/agents-health` | 过滤 `liveness != online` |
| 在线Agent | `GET /dashboard/summary` | `agents.active / agents.total` |
| 告警列表 | `GET /dashboard/cert-alerts` | 合并 external + agent 告警 |
| 趋势数据 | `GET /dashboard/certs-expiry?days=90` | 按天聚合为折线图 |
| 证书列表(外部) | `GET /external-certs` | 已有 |
| 证书列表(已部署) | `GET /agents/:id/detail` | 现有 N+1 问题，考虑新增聚合接口 |
| Agent 列表 | `GET /agents` | 已有，含 liveness/cert_count |
| Agent 详情 | `GET /agents/:id` + `GET /agents/:id/detail` | 已有 |
| 审计时间线 | `GET /audit` | 已有，`limit=10` |

**注意**: 证书管理页需同时展示外部证书和已部署证书，目前需要两个接口分别获取。可后续优化为统一接口，但首版前端可并行请求合并展示。
