测试原则：真实 Docker / OrbStack / minikube 环境；真实 Control API、真实 Python Agent、真实 nginx、真实 openssl 证书、真实 Kubernetes API；不使用 mock。

## 结论

本轮指定场景测试通过。

已验证：

- 多台 nginx 机器：3 个独立 edge 容器接入同一个控制平面。

- Agent 信息获取：`/agents`、`/agents/{id}/detail`、`/agents/{id}/assignments` 均返回真实在线状态、路径和 assignment 数。

## 真实测试中发现的问题

1. 本地 smoke `.env` 曾使用非法 Fernet key。mock 测试不会暴露，真实上传私钥时会在 `encrypt_key` 抛错并导致 `POST /external-certs` 500。已在本地 ignored `.env` 更换为合法 Fernet key。
2. minikube 清理 namespace 后，历史 hostpath PostgreSQL 数据可能残留，导致新 Secret 的 `POSTGRES_PASSWORD` 和旧数据库密码不一致。已通过清理 minikube hostpath 后重建解决。
3. 初版 Docker 场景脚本曾先部署 45 天证书，再尝试用 10 天旧证书覆盖；Agent 按设计没有降级更新。脚本已改为 edge-b 专门执行“旧证书 -> 新证书”的正向续期替换。
