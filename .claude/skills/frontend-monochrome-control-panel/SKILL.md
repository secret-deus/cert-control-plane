# Frontend Monochrome Control Panel Skill

用于生成和约束一套统一的后台设计语言，目标不是通用 SaaS 后台，而是接近参考图那种：

- 黑白灰为绝对主色
- 只用少量亮色做重点强调
- 大圆角但不软塌
- 阴影弱，组件更像嵌在页面里
- 留白克制，信息密度高

## 适用范围

- 监控看板
- 证书资产列表
- Agent 舰队管理
- 系统设置页
- 登录页
- 侧栏 / 顶栏 / 卡片 / 表格 / 图表 / 弹层

---

## 核心设计语言

### 1. 颜色原则

页面主体只允许使用以下颜色逻辑：

- **主背景**：黑灰色系
- **面板背景**：比主背景略亮一层的深灰
- **边框**：非常轻的白色透明边框
- **文字**：白 / 浅灰 / 中灰三级
- **强调色**：
  - 橙色：告警、主按钮、趋势主线、关键操作
  - 浅绿色：健康、成功、On track、通过状态

### 2. 禁止事项

禁止以下视觉表达：

- 大面积蓝色、紫色、青绿色块状背景
- 一页里出现 4 种以上强强调色
- 玻璃拟态、重阴影、浮空卡片感
- 彩色面板背景作为主视觉
- 过亮的描边和高饱和霓虹效果

### 3. 推荐色板

基础中性色：

- `#101010` 页面底色
- `#141414` 主工作区
- `#181818` 面板底色
- `#F5F5F5` 一级文字
- `#B3B3B3` 二级文字
- `#8C8C8C` 三级文字
- `rgba(255,255,255,0.06)` 面板边框

强调色：

- `#FF995C` 橙色主强调
- `#73BF69` 浅绿色健康态
- `rgba(255,153,92,0.10)` 橙色浅底
- `rgba(115,191,105,0.10)` 绿色浅底

## 布局规范

### 1. 外层壳子

使用真实工作区布局，不做作品展示式舞台：

- 页面直接全屏铺开
- 左侧固定深色导航栏
- 右侧主内容区独立滚动
- 不要在页面四周保留大块无意义灰色留白

### 2. 顶栏

顶栏结构固定为：

- 面包屑 / 小标题
- 大页面标题
- 右侧少量控制项（如告警、头像）

不要在顶栏放过多筛选器和复杂说明。

### 3. 内容分区

优先使用以下结构：

1. 顶部一排 KPI 小卡
2. 中间一张主图卡
3. 下方两张业务卡并排

不要把首页做成一整页碎卡片拼贴墙。

---

## 组件视觉规范

### 1. 圆角

圆角必须统一分层，不可随意取值：

- 页面主卡：`18px` 到 `22px`
- 次级区块：`16px` 到 `18px`
- 按钮 / 输入框：`14px` 到 `18px`
- badge / pill：`12px` 到 `16px`

原则：

- 可以圆，但不能“玩具化”
- 不能使用过度圆润的胶囊感容器充满整页

### 2. 阴影

阴影必须非常弱，优先使用：

- 轻边框
- 轻内阴影
- 面板与底色的明度差

不要使用：

- 大范围投影
- 漂浮卡片效果
- 多层发光阴影

### 3. 边框

以边框建立层级，而不是靠彩色底块：

- 主卡：`1px rgba(255,255,255,0.06)`
- 次级区：更浅的白色透明边框
- hover/active：仅略微提升背景和边框透明度

### 4. 图标

图标颜色策略：

- 默认：白或浅灰
- 状态相关：才允许橙 / 绿

不要把每张卡的图标都做成不同颜色主题。

---

## 图表规范

### 1. 图表角色

图表必须承担信息表达，不是装饰。

优先顺序：

- 主趋势图：页面主视觉
- 分布图 / 环图：辅助判断
- 表格：承接具体对象和动作

### 2. 图表配色

图表最多使用 3 个层级：

- 主线：橙色
- 次线 / 基线：白或浅灰
- 健康提示：浅绿色

不要使用彩虹色图表。

### 3. 图表容器

图表外层必须有独立内框：

- 深灰底
- 轻边框
- 稍大圆角
- 上方有简洁标题和少量标记

---

## 表格规范

表格必须接近运维控制台而非 CRM：

- 行高偏紧
- 表头小写/轻量 uppercase
- 行 hover 仅轻微提亮
- 选中行只做弱背景变化
- 动作按钮尽量文字化，不要满屏彩色按钮

风险、状态、健康度这类字段：

- 用小 badge 表示
- 非必要不要整行染色

---

## 页面类型落地规则

### 1. Dashboard

- 3 到 4 张 KPI 足够
- 1 张主图卡承担主视觉
- 1 张告警表
- 1 张健康状态卡
- 页面节奏要像“看板”，不是“组件展览”

### 2. 列表页（证书 / Agent）

- 上方只保留必要概览卡和筛选区
- 中部用大表格承担主体
- 右侧用详情抽屉 / dossier 承担深度信息
- 不要在列表页再复制 Dashboard 的图表密度

### 3. 设置页

- 用中性卡片和只读摘要块
- 不要给每一类设置一个强彩色主题

### 4. 登录页

- 仍然要沿用同一套黑白灰语言
- 登录卡和说明卡并列
- 主操作按钮用橙色

---

## 文字与排版

推荐字体：

- `Manrope`

排版原则：

- 标题粗但不夸张
- 辅助文字尽量短
- 大数字用于 KPI
- 正文说明只做必要补充

建议层级：

- 页面主标题：`32px` 到 `40px`
- 卡片标题：`18px` 到 `22px`
- 普通正文：`13px` 到 `14px`
- kicker：`10px` 到 `11px` uppercase

---

## 实现检查清单

在任何页面改造完成后，必须自查：

1. 主色是否仍是黑白灰
2. 强调色是否只在关键位置出现
3. 是否存在多余的彩色底卡
4. 圆角是否统一分层
5. 阴影是否过重
6. 表格是否过松散
7. 图表是否承担了真正信息表达
8. 首页是否避免了碎卡堆砌

```text

2. 强调色只保留橙色和浅绿色
3. 圆角和阴影克制，接近现代运维控制台
4. 不要做成普通 SaaS 后台卡片风
5. 优先参考当前项目 Layout / Dashboard 已有语言
```
