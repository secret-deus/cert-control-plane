# 安全审计清单

本文档提供 Cert Control Plane 的安全审计检查清单，用于确保系统在生产环境中的安全性。

## 🔐 认证与授权

### API 认证

- [ ] **Admin API Key**
  - [ ] 使用强随机数生成（至少 256 位）
  - [ ] 不在代码仓库中硬编码
  - [ ] 定期轮换（建议每 90 天）
  - [ ] 通过安全渠道分发

- [ ] **Agent Token**
  - [ ] 每个 Agent 独立 token
  - [ ] Token 使用加密存储
  - [ ] 支持 token 撤销
  - [ ] Token 有过期机制（可选）

### 路径与来源隔离

- [ ] **`/api/control/*`（Control API）**
  - [ ] 仅限运维网络访问
  - [ ] 需要 Admin API Key

- [ ] **`/api/agent/*`（Agent API）**
  - [ ] 仅限 Agent 节点 IP 访问
  - [ ] 需要 Agent Token
  - [ ] 注册端点配合来源限制、限流和审计

### 访问控制

- [ ] 实施最小权限原则
- [ ] Agent 只能访问自己的证书
- [ ] Admin API 操作记录审计日志
- [ ] 敏感操作需要二次确认（可选）

## 🔒 数据加密

### 传输加密

- [ ] **TLS 配置**
  - [ ] 使用 TLS 1.2 或更高版本
  - [ ] 禁用弱密码套件
  - [ ] 启用 HSTS
  - [ ] 配置证书链完整

- [ ] **密码套件推荐**
  ```
  ECDHE-ECDSA-AES128-GCM-SHA256
  ECDHE-RSA-AES128-GCM-SHA256
  ECDHE-ECDSA-AES256-GCM-SHA384
  ECDHE-RSA-AES256-GCM-SHA384
  ```

### 存储加密

- [ ] **私钥加密**
  - [ ] 使用 Fernet 对称加密
  - [ ] 密钥通过 HSM 或密钥管理服务存储
  - [ ] 定期轮换加密密钥（建议每年）
  - [ ] 密钥备份安全存储

- [ ] **数据库加密**
  - [ ] 启用 PostgreSQL TDE（如可用）
  - [ ] 敏感字段加密存储
  - [ ] 数据库连接使用 SSL

- [ ] **文件系统加密**
  - [ ] Agent 私钥文件权限 0600
  - [ ] Agent token 文件权限 0600
  - [ ] 考虑使用文件系统加密（LUKS、dm-crypt）

## 🛡️ 输入验证

### API 输入

- [ ] **请求数据验证**
  - [ ] 使用 Pydantic 模型验证
  - [ ] 验证字段类型和格式
  - [ ] 限制字符串长度
  - [ ] 验证文件上传类型和大小

- [ ] **防止注入攻击**
  - [ ] 使用参数化查询（SQLAlchemy ORM）
  - [ ] 不拼接 SQL 语句
  - [ ] 验证文件路径（防止目录遍历）
  - [ ] 验证证书内容格式

### 文件操作

- [ ] **证书文件处理**
  - [ ] 验证 PEM 格式
  - [ ] 提取并验证证书元数据
  - [ ] 限制文件大小
  - [ ] 不执行外部命令处理证书

## 🔍 日志与审计

### 审计日志

- [ ] **审计事件记录**
  - [ ] Agent 注册、审批、拒绝、删除
  - [ ] 证书上传、分配、删除
  - [ ] Rollout 创建、启动、暂停、恢复、回滚
  - [ ] API 访问日志

- [ ] **审计日志保护**
  - [ ] 日志不可篡改
  - [ ] 日志包含时间戳、操作者、IP 地址
  - [ ] 日志保留至少 90 天
  - [ ] 日志备份到安全位置

### 应用日志

- [ ] **日志安全**
  - [ ] 不记录敏感信息（密钥、token）
  - [ ] 使用结构化日志格式
  - [ ] 日志级别配置合理
  - [ ] 日志轮转配置

## 🚨 错误处理

### 错误信息

- [ ] **错误响应**
  - [ ] 不泄露内部实现细节
  - [ ] 使用通用错误消息
  - [ ] 记录详细错误到日志
  - [ ] 区分开发和生产环境错误

### 异常处理

- [ ] **全局异常处理**
  - [ ] 捕获所有未处理异常
  - [ ] 返回 500 错误而非暴露堆栈
  - [ ] 记录异常详情到日志
  - [ ] 监控异常频率

## 🌐 网络安全

### 防火墙规则

- [ ] **入站规则**
  ```
  443/TCP  - 运维网络 / Agent 节点 -> 外部 HTTPS 入口
  8000/TCP - 外部网关 -> FastAPI 应用
  22/TCP   - 管理网络 -> 所有服务器
  ```

- [ ] **出站规则**
  ```
  443/TCP  - 控制平面 -> 外部 API（如需要）
  5432/TCP - 应用服务器 -> 数据库服务器
  ```

### DDoS 防护

- [ ] 实施速率限制
- [ ] 配置连接限制
- [ ] 使用负载均衡器
- [ ] 监控异常流量

## 📦 依赖安全

### 依赖管理

- [ ] **依赖扫描**
  - [ ] 定期运行 `pip audit`
  - [ ] 使用 GitHub Dependabot
  - [ ] 监控 CVE 数据库
  - [ ] 及时更新有漏洞的依赖

- [ ] **依赖锁定**
  - [ ] 锁定依赖版本
  - [ ] 使用 hash 验证
  - [ ] 从官方源安装

### 容器安全

- [ ] **Docker 镜像**
  - [ ] 使用官方基础镜像
  - [ ] 定期更新基础镜像
  - [ ] 扫描镜像漏洞
  - [ ] 不使用 root 用户运行

## 🔑 密钥管理

### 密钥生命周期

- [ ] **密钥生成**
  - [ ] 使用加密安全随机数生成器
  - [ ] 密钥长度符合标准（AES-256, RSA-2048+）
  - [ ] 在安全环境中生成

- [ ] **密钥存储**
  - [ ] 使用 HSM 或密钥管理服务
  - [ ] 分离密钥和数据
  - [ ] 访问权限严格控制
  - [ ] 加密备份

- [ ] **密钥轮换**
  - [ ] 制定轮换计划
  - [ ] 测试轮换流程
  - [ ] 保留旧密钥备份（过渡期）
  - [ ] 文档化轮换记录

## 🔄 备份与恢复

### 数据备份

- [ ] **备份策略**
  - [ ] 定期自动备份（每日）
  - [ ] 备份加密存储
  - [ ] 异地备份
  - [ ] 测试恢复流程

- [ ] **备份内容**
  - [ ] 数据库数据
  - [ ] 审计日志
  - [ ] 配置文件
  - [ ] 加密密钥（单独备份）

### 灾难恢复

- [ ] **恢复计划**
  - [ ] 制定详细恢复步骤
  - [ ] 定期演练
  - [ ] 恢复时间目标（RTO）
  - [ ] 恢复点目标（RPO）

## 📋 合规性检查

### 数据保护

- [ ] **GDPR/个人信息保护**
  - [ ] 最小化数据收集
  - [ ] 数据保留期限
  - [ ] 数据删除机制
  - [ ] 数据导出功能

### 行业标准

- [ ] **安全标准**
  - [ ] OWASP Top 10 检查
  - [ ] PCI-DSS（如处理支付）
  - [ ] SOC 2（如适用）
  - [ ] ISO 27001（如适用）

## 🧪 安全测试

### 渗透测试

- [ ] **测试范围**
  - [ ] API 端点测试
  - [ ] 认证绕过测试
  - [ ] 权限提升测试
  - [ ] 注入攻击测试

- [ ] **测试频率**
  - [ ] 发布前完整测试
  - [ ] 每季度常规测试
  - [ ] 重大变更后测试

### 漏洞扫描

- [ ] **扫描工具**
  - [ ] 使用自动化扫描工具
  - [ ] 扫描 Web 应用漏洞
  - [ ] 扫描依赖漏洞
  - [ ] 扫描配置问题

## 📞 事件响应

### 响应流程

- [ ] **事件分类**
  - P1: 数据泄露、系统入侵
  - P2: 服务中断、认证绕过
  - P3: 性能问题、可疑活动

- [ ] **响应步骤**
  1. 检测和确认
  2. 遏制和隔离
  3. 根因分析
  4. 修复和恢复
  5. 事后总结

- [ ] **联系方式**
  - [ ] 安全团队联系方式
  - [ ] 紧急联系流程
  - [ ] 外部安全专家联系方式

### 月度检查

- [ ] 检查异常日志
- [ ] 审查访问日志
- [ ] 验证备份完整性
- [ ] 检查依赖漏洞

### 季度检查

- [ ] 审查用户权限
- [ ] 测试备份恢复
- [ ] 审查审计日志
- [ ] 检查密钥轮换计划

### 年度检查

- [ ] 完整渗透测试
- [ ] 依赖安全审计
- [ ] 密钥轮换执行
- [ ] 灾难恢复演练
- [ ] 安全培训更新

## 📊 安全指标

跟踪以下安全指标：

| 指标 | 目标 | 说明 |
|------|------|------|
| 认证失败率 | < 5% | 认证失败占总请求比例 |
| 异常访问检测 | < 10/天 | 检测到的异常访问次数 |
| 漏洞修复时间 | < 7天 | 高危漏洞修复平均时间 |
| 备份成功率 | 100% | 备份任务成功率 |
| 安全事件 | 0 P1 | P1 级别安全事件数量 |
